Privacy Policy

Focus Time Tracker Chrome Extension

Effective Date:

Executive Summary

Focus Time Tracker is a privacy-first Chrome extension designed to help users track website usage and maintain focus during work sessions. We are committed to protecting your privacy through local-first data storage, minimal data collection, and comprehensive user control mechanisms.

Key Privacy Principles

  • Local-First Storage: All data is stored locally on your device using Chrome's secure storage APIs
  • No Data Sales: We never sell, rent, or share your personal data with third parties
  • Minimal Collection: We only collect data necessary for core functionality
  • User Control: Complete control over your data with export and deletion options
  • Transparency: Open-source approach with clear documentation of all data practices

Detailed Data Collection and Usage

Website Usage Data

Data Collected:

Domain names, time spent on sites, visit counts, last visit timestamps, and page focus/blur events.

Storage Key:

dailyStats_[YYYY-MM-DD] - Organized by date for efficient retrieval and cleanup

Retention:

30 days (configurable), with automatic cleanup of expired data

Purpose:

Provides productivity insights and enables focus mode functionality

User Activity Detection

Data Collected:

Mouse movements, clicks, keyboard inputs, scroll events, tab focus/blur events, and system sleep/wake detection.

Processing:

Processed in real-time to determine active vs. inactive status. No content or specific keystrokes are recorded.

Purpose:

Ensures accurate time tracking by only counting active usage time

Deep Focus Sessions

Data Collected:

Session start/end times, duration, unique session IDs, completion status, and blocked sites during sessions.

Storage Key:

deepFocusSessions_[YYYY-MM-DD] - Date-organized for analytics and cleanup

Purpose:

Tracks focus session effectiveness and provides productivity analytics

Site Blocking and Override Data

Data Collected:

Blocked domain lists, override session records with timestamps, duration, and reason for temporary access.

Storage Keys:

blockingManagerState, overrideSessions

Purpose:

Implements focus mode blocking and tracks temporary access for productivity analysis

User Preferences and Settings

Data Collected:

Extension configuration, theme preferences, notification settings, and focus mode preferences.

Storage Key:

settings, extensionConfig

Purpose:

Personalizes user experience and maintains consistency across browser sessions

Optional User Account Data

Data Collected (When Sync Enabled):

User ID, email address, display name, and authentication tokens for Firebase integration.

Storage Key:

userInfo

Purpose:

Enables cross-device synchronization with our web application (optional feature)

Data Processing and Security

Local Data Storage

All data is stored locally on your device using Chrome's secure storage APIs (chrome.storage.local). This ensures:

  • Data remains under your control
  • No external servers can access your information
  • Data is encrypted at rest by Chrome's security mechanisms
  • Automatic cleanup when extension is uninstalled

Security Measures

Technical Security Implementation

  • Input Validation: All user inputs are validated and sanitized
  • Context Security: Extension context invalidation detection and graceful recovery
  • Message Authentication: Origin validation for all inter-component communication
  • Timeout Protection: All asynchronous operations have timeout mechanisms
  • Content Security Policy: Strict CSP prevents code injection attacks
  • Error Boundaries: Comprehensive error handling prevents data corruption

Data Transmission Security

When optional sync is enabled:

  • All data transmitted over HTTPS with TLS 1.3
  • Authentication via Firebase secure tokens
  • Data encryption in transit and at rest
  • User can disable sync at any time

Chrome Extension Permissions

Our extension requests specific permissions to provide core functionality. Each permission is used exclusively for its stated purpose:

Permission Purpose Data Access
storage Store user preferences and tracking data locally Local storage only - no external access
tabs Monitor tab changes for accurate time tracking Tab URLs and focus state - processed locally
activeTab Identify current website for tracking and blocking Current tab URL only - no content access
declarativeNetRequest Block distracting websites during focus mode Block/allow decisions - no data collection
webNavigation Track page navigation for accurate time measurement Navigation timestamps - no page content
favicon Display website icons in the interface Icon URLs only - for UI enhancement
<all_urls> Inject content scripts for activity detection User activity events - no page content

Important Note on <all_urls> Permission

This permission is required to inject content scripts that detect user activity (mouse movements, clicks, scrolling) across all websites. This enables accurate time tracking by distinguishing between active and inactive usage. We do not access, read, or store any website content, form data, or personal information from web pages.
